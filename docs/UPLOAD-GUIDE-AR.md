# دليل رفع ومعاينة ثيم Lumina في سلة 🚀

هذا الدليل يشرح **خطوة بخطوة** كيف ترفع ثيم Lumina وتعاينه ثم تنشره في متجر سلة، اعتماداً على إطار **Salla Twilight** والأداة الرسمية **Salla CLI**.

> المرجع الرسمي: https://docs.salla.dev/

---

## 🧩 المتطلبات قبل البدء

| الأداة | الإصدار | للتحقق |
|--------|---------|--------|
| Node.js | **22.12+ (LTS) — أو 20.19+ على الأقل** | `node -v` |
| pnpm | 8+ (سلة تفرض pnpm) | `npm i -g pnpm` ثم `pnpm -v` |
| Salla CLI | الأحدث | `npm i -g @salla.sa/cli` ثم `salla -v` |
| حساب مطوّر سلة | — | https://salla.partners |

> ⚠️ **انتبه لإصدار Node:** أداة Salla CLI الحديثة تحتوي وحدات «ESM فقط». دعم `require()` لها أُضيف في **Node v20.19.0** واكتمل في **v22.12.0**. أي إصدار أقدم (مثل v20.18.x) سيُسبب الخطأ `ERR_REQUIRE_ESM` عند تشغيل أوامر سلة. استخدم Node 22 LTS لتفادي المشكلة تماماً.

## الخطوة 1 — فك الضغط
فك ضغط الملف `lumina-theme.zip`. سيظهر مجلد `lumina-theme/` يحتوي على:
```
twilight.json   ← إعدادات الثيم والمكونات
package.json    ← الاعتمادات وأوامر البناء
tailwind.config.js / postcss.config.js / webpack.config.js
src/            ← القوالب (twig) والأصول (js/scss/images) والترجمات
docs/           ← التوثيق
```

## الخطوة 2 — تثبيت الاعتمادات
من داخل مجلد الثيم:
```bash
cd lumina-theme
pnpm install --frozen-lockfile
```
> ⚠️ **مهم جداً:** استخدم `--frozen-lockfile` ليُثبّت **نفس الإصدارات المختبَرة** من ملف `pnpm-lock.yaml` المرفق. هذا يمنع خطأ `ERR_REQUIRE_ESM` (الذي يحدث عندما يجلب npm/pnpm نسخة أحدث «ESM فقط» من حزمة مثل `glob`).
>
> الإصدارات في `package.json` **مثبّتة بدون `^`** عمداً لنفس السبب — لا تغيّرها ولا تشغّل `pnpm update`.

## الخطوة 3 — بناء الأصول (CSS/JS)
```bash
pnpm run production
```
سينشئ مجلد `public/` يحتوي على `app.css`, `lumina.js`, `home.js`, `product.js` … إلخ.
> أثناء التطوير استخدم `pnpm run watch` لإعادة البناء تلقائياً.

## الخطوة 4 — تسجيل الدخول إلى سلة
```bash
salla login
```
سيفتح المتصفح لتأكيد الدخول بحساب المطوّر.

## الخطوة 5 — ربط الثيم (أول مرة فقط)
```bash
salla theme link
```
اختر المتجر التجريبي (Demo Store) الذي تريد المعاينة عليه.
> إن لم يكن لديك متجر تجريبي، أنشئه من لوحة **Salla Partners → Themes**.

## الخطوة 6 — المعاينة المباشرة 👀
```bash
salla theme preview
```
- ستحصل على رابط معاينة مباشر (Live Preview) يعكس تعديلاتك فوراً.
- افتح الرابط على الجوال والكمبيوتر لاختبار التجربة المتجاوبة.

## الخطوة 7 — رفع الثيم إلى سلة
```bash
salla theme push
```
يرفع الثيم إلى حسابك في **Salla Partners**.

## الخطوة 8 — النشر / الإصدار
```bash
salla theme publish
```
ثم من لوحة **Salla Partners → Themes** أكمل:
1. تعبئة بيانات الثيم (الاسم، الوصف، الصور، السعر).
2. رفع لقطات الشاشة (Screenshots).
3. إرسال الثيم للمراجعة (UI/UX + Technical + Metadata).

## 🎨 تخصيص الثيم من لوحة التاجر (بدون كود)
بعد التفعيل، يدخل التاجر إلى:
**متجري → المظهر (Theme) → تخصيص**
ويجد كل الخيارات التالية جاهزة (معرّفة في `twilight.json`):

- الطباعة (خط العناوين + الخطوط العربية/Google)
- الألوان واللون المساعد + **الوضع الداكن**
- انحناء الزوايا، عرض المحتوى، المسافات بين الأقسام
- خيارات الهيدر (تثبيت، شريط إعلاني، بحث فوري)
- تجربة الجوال (شريط سفلي، زر شراء ثابت)
- بطاقات المنتجات (إضافة سريعة، صورة عند المرور)
- صفحة المنتج (تكبير، شحن مجاني، تقدير التوصيل)
- السلة (حد الشحن المجاني + شريط التقدم)
- الفوتر (وضع داكن، شارات ثقة، نشرة بريدية)
- الحركات والأداء

كما يمكن للتاجر **إعادة ترتيب أقسام الصفحة الرئيسية** وإظهار/إخفاء أي مكوّن عبر السحب والإفلات من محرّر الثيم.

### 🎭 الأنماط الجاهزة (Presets)
في أعلى إعدادات الثيم يوجد خيار **«نمط جاهز / Global preset»** يضبط اللون المساعد وانحناء الزوايا بنقرة واحدة:
- زمرّدي فاخر (افتراضي) · أسود وذهبي · وردي هادئ · أزرق ملكي · بيج اسكندنافي · بنفسجي عصري
- اختر **«مخصص / Custom»** للتحكم اليدوي الكامل باللون المساعد والزوايا.
> اللون الأساسي للمتجر يبقى دائماً من خاصية ألوان سلة الرسمية (يُحترم لون علامتك التجارية).

### الصفحات المدعومة في الثيم
الرئيسية · التصنيف/القائمة · المنتج · السلة · صفحة الشكر · المدونة (قائمة + مقال) · الماركات (قائمة + ماركة) · صفحة الهبوط · الولاء/المكافآت · صفحة تعريفية · حساب العميل (الملف الشخصي، الطلبات + التفاصيل، المفضلة، الإشعارات).

## 🛠️ حل المشكلات الشائعة
| المشكلة | الحل |
|--------|------|
| `only-allow pnpm` خطأ | استخدم `pnpm install` وليس `npm install` |
| لا تظهر التنسيقات | تأكد من تنفيذ `pnpm run production` ووجود `public/app.css` |
| البحث/السلة لا تعمل في المعاينة | هذه مكونات Salla Web Components تعمل فقط داخل بيئة سلة الحقيقية (المعاينة عبر `salla theme preview`) |
| الأيقونات لا تظهر | تأكد من اتصال الإنترنت لتحميل `sallaicons.css` من CDN سلة |
| `Cannot find module 'glob'` أثناء البناء | تأكد من وجود `glob` ضمن devDependencies (مضافة بالفعل) ثم أعد `pnpm install --frozen-lockfile` |
| `The 'leading-12' class does not exist` | لا تحذف مفاتيح `extend` في `tailwind.config.js` فهي مطلوبة من safe-list الخاص بـ Twilight |
| **`ERR_REQUIRE_ESM` (require() of ES Module)** | السبب: تم جلب نسخة أحدث «ESM فقط» من حزمة (غالباً `glob` v11+). **الحل:** احذف `node_modules` و `pnpm-lock.yaml` المحلي القديم، ثم نفّذ `pnpm install --frozen-lockfile` لاستخدام ملف القفل المرفق. تأكد أن `package.json` يحتوي إصدارات مثبّتة بدون `^` وقسم `pnpm.overrides.glob = "10.5.0"`. لا تستخدم `npm install` مطلقاً (سلة تفرض pnpm). |

> ✅ تم اختبار البناء فعلياً: `pnpm install` ثم `pnpm run production` يكتمل بنجاح وينتج مجلد `public/` (app.css + جميع ملفات JS). التحذير الوحيد هو حجم `app.css` وهو طبيعي لأنه يتضمن أنماط مكوّنات سلة الكاملة (نفس سلوك الثيم الرسمي «رائد»).

## 🚑 خطأ `ERR_REQUIRE_ESM` عند تشغيل أوامر Salla CLI

إذا ظهر لك خطأ مثل:
```
Error [ERR_REQUIRE_ESM]: require() of ES Module
C:\Users\<اسمك>\AppData\Local\nvm\v20.18.3\node_modules\...
  code: 'ERR_REQUIRE_ESM'
```
فالمسار يحتوي `\nvm\v20.18.3\node_modules\` (مجلد الحزم العامة) → **المشكلة في إصدار Node + Salla CLI على جهازك، وليست في الثيم.**

**السبب:** أداة سلة تستدعي وحدات «ESM فقط» بـ `require()`، وهذا غير مدعوم في Node أقدم من v20.19.

**الحل (الموصى به) — ترقية Node وإعادة تثبيت CLI:**
```powershell
nvm install 22.12.0
nvm use 22.12.0
node -v                       # تأكد: v22.12.0
npm uninstall -g @salla.sa/cli
npm cache clean --force
npm install -g @salla.sa/cli  # أعد التثبيت على إصدار Node الجديد (ضروري)
salla -v
```

**بديل سريع (حل مؤقت دون ترقية):**
```powershell
set NODE_OPTIONS=--experimental-require-module
salla theme preview
```

> ملاحظة: بعد تغيير إصدار Node عبر nvm، **يجب إعادة تثبيت Salla CLI** لأن الحزم العامة مرتبطة بكل إصدار Node على حدة.

> ملاحظة: ملف المعاينة داخل بيئة Arena يعرض شكل التصميم فقط بدون بيانات سلة الحية؛ التجربة الكاملة تظهر عبر `salla theme preview`.
